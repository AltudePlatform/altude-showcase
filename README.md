## Submit a Project

Built something with Altude? We’d love to feature it.

You can submit:

- A customer project
- A public repository
- A demo
- A starter template
- An integration
- A tutorial or walkthrough
- A case study

### Submission Guidelines

Please include:

- **Project name**
- **Short description**
- **Repository, demo, website, or case study link**
- **Project type**: customer project, demo, template, integration, tutorial, etc.
- **Primary language or stack**
- **Use case**
- **Project owner or organization**
- **Approval status**, if this is a customer project
- Optional: screenshot, logo, video, blog post, or case study link

### Customer Project Requirements

Customer projects must have explicit approval before being listed.

Before submitting a customer project, confirm that:

- The customer has approved being named.
- The project URL is public, or the public summary has been approved.
- Any screenshots are approved and do not expose private data.
- Any logos, quotes, or metrics are approved for use.
- Any claims, statistics, or results are accurate and approved.
- The description matches approved public messaging.

If the customer cannot be named, use an anonymized format such as:

```markdown
### AI Workflow Assistant

A production workflow assistant built with Altude for a B2B SaaS team.

- **Customer:** Private / undisclosed
- **Use case:** Internal operations automation
- **Industry:** SaaS
- **Status:** Public summary approved
```

## Suggested Project Entry Format

Use this format when adding a public project:

```markdown
### Project Name

One- or two-sentence description of the project and how it uses Altude.

- **Repository:** https://github.com/example/project
- **Demo:** https://example.com
- **Type:** Demo / template / integration / playground / tutorial
- **Language:** TypeScript
- **Stack:** Next.js, Node.js, React, etc.
- **Use case:** AI assistant, workflow automation, data operations, etc.
- **Status:** Public example
```

Use this format when adding an approved customer project:

```markdown
### Customer Project Name

Approved description of what the customer built with Altude.

- **Customer:** Customer Name
- **Use case:** Workflow automation / AI assistant / data operations / etc.
- **Industry:** SaaS / healthcare / finance / education / etc.
- **Link:** https://example.com
- **Case study:** https://example.com/case-study
- **Status:** Public / case study / private summary approved
```
